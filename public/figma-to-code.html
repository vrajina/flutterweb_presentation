Figma → Code через AI IDE

Исследование best practices

Актуально на июнь 2026. Источники: builder.io, urbaninsight.com, monterail.com, sixtythirtyten.co, документация Figma.

Ключевой сдвиг 2025–2026: от скриншотов к структурированным данным

Раньше AI получал скриншот макета и «угадывал» отступы, цвета, иерархию компонентов.

Сейчас золотой стандарт — Model Context Protocol (MCP): AI-агент в IDE читает реальные данные из Figma — токены, flex-свойства, типографику, структуру компонентов. Это радикально снижает «галлюцинации» и ручную правку.

01 Подготовка дизайна в Figma

⚠ Важно
Garbage in — garbage out. Никакой AI не сделает хороший код из грязного файла Figma. Подготовка дизайна — это 50% успеха.
ТребованиеЗачем
Auto LayoutAI понимает адаптивность и связи между элементами. Без этого — абсолютное позиционирование и невалидный CSS
Design Tokens (Variables)Цвета, отступы, типографика через переменные. Иначе AI выдаст захардкоженные HEX-значения
Компонентная архитектураПереиспользуемые паттерны (карточки, хедеры, кнопки). AI лучше работает с целыми компонентами
Семантические имена слоёвPrimaryButton вместо Frame 123. Прямо влияет на читаемость кода
Логическая группировкаГруппы = границы компонентов. Помогает AI генерировать модульный код

02 Figma MCP — основной инструмент связки

Figma MCP Server — мост между IDE и данными дизайна. AI-агент читает не пиксели, а реальную структуру: padding, flex, типографику, варианты компонентов.

Настройка в Cursor

1. Ctrl+Shift+P → Open Chat
2. Ввести: /add-plugin figma
3. Cursor Settings → Tools & MCP → Connect (авторизация Figma)

Настройка в VS Code

Создать файл .vscode/mcp.json:

{
  "inputs": [],
  "servers": {
    "figma": {
      "url": "https://mcp.figma.com/mcp",
      "type": "http"
    }
  }
}

Нажать Start → авторизоваться через браузер.

Требования к аккаунту

Нужен Dev Seat или Full Seat на плане Professional / Organization / Enterprise.

03 Figma Code Connect — маппинг компонентов

Code Connect связывает компоненты Figma с реальными компонентами в коде. Когда AI видит Button в Figma, он не генерирует новую кнопку, а использует существующий <Button variant="primary" />.

СпособДля когоПлюсыМинусы
CLIРазработчикиПолный контроль, маппинг сложных пропсовНужно писать template-файлы
UIКомандыБыстрая настройка, несколько фреймворковНет live-сниппетов в Inspect
  • Начинать с базовых компонентов: Button, Input, Card, Badge
  • Единые имена: пропсы в коде = свойства в Figma
  • Поддерживать актуальность: маппинги = живая документация

07 Практические рекомендации

Минимальный набор для старта

  1. Figma (Professional план + Dev Seat)
  2. Cursor IDE (или VS Code с GitHub Copilot)
  3. Figma MCP — подключение за 5 минут

Чеклист подготовки Figma-файлов для дизайнеров

  • Все компоненты используют Auto Layout
  • Цвета, отступы, шрифты — через Variables (не захардкожены)
  • Слои названы семантически (ProductCard, не Frame 47)
  • Компоненты переиспользуемые, с вариантами
  • Группировка отражает логическую структуру UI
